Background

Geological literature often contains tables of fossil or lithology observations with stratigraphic period/stage names but rarely specifies which time scale version was used. The GTSKG knowledge base encodes boundary ages for over 100 versions (1879–2024), which can be used as a lookup oracle to match period/stage names to version-specific numeric ages. Combined with spatial (lat/lon) and lithological features, a machine learning model can be trained to automatically classify legacy records into the correct time scale version.

Methodology

1
Raw Data Parsing
Parse Excel records with Period, Stage, LMU, Lithology, Lat/Lon, Country, Continent, and Publication Year
2
KG Boundary Lookup
Query GTSKG TTL files by version + period/stage name to retrieve numeric age boundaries and uncertainties
3
Feature Encoding
Label-encode categorical features (Country, Continent, LMU, Period, Stage, Lithology); KNN-impute missing values
4
ML Classification
Train K-Nearest Neighbors (k=1) on encoded features; predict the time scale version for test records

Key Code

# Query KG for boundary ages of a period/stage in a specific version
def get_boundary_value(version_value, label):
    graph = Graph()
    graph.parse(f'{version_value}.ttl', format='ttl')
    start = graph.value(subject=URIRef(f'utf#{label}'), predicate=URIRef('utf#start'))
    end   = graph.value(subject=URIRef(f'utf#{label}'), predicate=URIRef('utf#end'))
    return start, end
